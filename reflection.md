# Reflection: Building a Persona-Based AI Chatbot

## What Worked Well
The most successful aspect of this project was adopting the OpenRouter API to centralize our requests, which allowed us to easily plug in models like Google's Gemini that understand nuanced personality traits. Structuring the system prompts using the `prompts.md` guidelines proved exceptionally effective. By giving the model a distinct background, core philosophies, constraints, and few-shot examples, the personas felt grounded and recognizable. Building a clean, modular React frontend (Vite) alongside a standard Node.js/Express backend made it easy to decouple the UI from the API integration, keeping concerns completely separated. State management inside React handled persona switching seamlessly, resetting the conversation thread exactly as required when a user decides to speak to a different personality.

## The GIGO Principle (Garbage In, Garbage Out)
Applying the GIGO principle deeply influenced how I approached the system prompts. Initially, it's easy to assume that just saying "You are Anshuman Singh" would yield a good enough imitation. However, without context, the LLM falls back to generic helpfulness. By passing high-quality, deeply researched input—such as the specific analogies of Airbnb or the undo feature for a Stack, plus enforcing a Chain-of-Thought (<thinking>) step—the output transformed from a generic assistant to a nuanced mentor. GIGO taught me that an AI is only as smart and characteristic as the constraints and context you actively feed it. Without few-shot examples of *how* these individuals actually speak, the tone would have been academic and disconnected from their real-world identities. 

## What I Would Improve
If I had more time, I would improve the contextual memory of the chatbot. Currently, each session restarts when a persona is switched. While this fits the requirements, a more advanced version might allow the three personas to maintain shared context so a student could ask Kshitij about DSA and then ask Abhimanyu about how that specific DSA topic applies to a job interview without having to repeat the topic. Additionally, I would implement streaming for the chat responses to make the UI feel even more interactive and real-time, reducing the perceived latency of waiting for a full completion. Lastly, expanding the suggestion chips dynamically based on the current context of the conversation rather than keeping them static would provide a richer user experience.
